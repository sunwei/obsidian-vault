# Friday 插件 Build in Public 运营策略

## 🎯 总体目标

1. **透明记录开发过程** → 让用户参与到 Friday 的成长里
    
2. **持续积累关注度** → 让潜在用户/合作伙伴看到 Friday 的价值
    
3. **教育和引导市场** → 让更多人知道「Obsidian 笔记可以一键变网站」
    
4. **构建核心用户群** → 培养早期用户，沉淀案例和口碑
    

---

## 🪜 三个层次的内容分类

### 1. 每日更新（轻量）

目的：保持活跃度和曝光  
形式：推特/X、社区群动态、朋友圈  
内容举例：

- 今天解决了某个 bug / 增加了一个小功能
    
- 一张动图演示「暗色模式切换」
    
- 分享一句产品感悟（例如「今天发现用户最怕的不是复杂，而是没教程」）
    
- 投票互动：「大家更想要 PDF 导出还是评论功能？」
    

---

### 2. 中期更新（周 / 月度）

目的：沉淀阶段成果，强化信任  
形式：Blog、知乎专栏、公众号、YouTube/B站  
内容举例：

- **功能更新日志**（Changelog）
    
- **案例分享**（Showcase，展示用户用 Friday 做的站点）
    
- **设计思路解读**（为什么要做双语支持、为什么选择免费+增值模式）
    
- **路线图进展**（Roadmap，对比完成与未完成部分）
    

---

### 3. 长期内容（季度 / 半年 / 年度）

目的：树立品牌和愿景，拓展市场影响力  
形式：长文复盘、线上发布会、社群直播  
内容举例：

- **里程碑总结**：用户数突破 1k、重大合作、产品重构
    
- **年度复盘**：总结做了哪些尝试、学到了什么
    
- **未来愿景**：Friday 想要成为怎样的产品，为谁服务
    
- **深度文章**：比如「为什么我们相信笔记就是未来的个人网站」
    

---

## 📚 教育与培训模块

### 1. 教程（基础教育内容）

- 快速上手指南（5 分钟发布网站）
    
- 功能点演示（主题切换、多语言内容、SEO 支持）
    
- 场景化案例（写作博主、英语学习者、团队协作）
    

### 2. 培训（进阶用户运营）

- 线上 Workshop：「从 0 到 1 搭建你的 Friday 站点」
    
- 系列课程：「Obsidian + Friday = 个人知识库 → 网站」
    
- 定期社群答疑/直播，和用户互动
    

👉 教程 = 拉新，培训 = 留存 & 提升粘性

---

## 📌 内容素材准备清单

- **截图 & 动态 GIF** → 每个功能点都要有配套视觉素材
    
- **用户案例库** → 收集 Showcase，作为固定栏目更新
    
- **公开路线图** → Trello/Notion Board，方便对照更新
    
- **开发日志** → GitHub issue / changelog 改写成运营文案
    
- **Workshop/直播录屏** → 二次剪辑，变成短视频/文章素材

## 📅 示例内容日历（4 周循环）

**第 1 周**

- 日更：功能小更新动图
    
- 周更：Changelog + 博客文章「本月我们修复了什么」
    

**第 2 周**

- 日更：开发感悟/投票
    
- 周更：用户 Showcase（某用户用 Friday 搭建英语学习网站）
    

**第 3 周**

- 日更：截图 + Bug 修复小故事
    
- 周更：功能设计思路文章（比如为什么要做免费基础版）
    

**第 4 周**

- 日更：轻量互动（比如“你们想要的新功能是？”）
    
- 月更：路线图进展 + 下月预告
    
- 举办一次 Workshop，产出视频/文章

✅ 总结：

- **每日**：碎片化 → 保持活跃度
    
- **中期**：阶段性总结 → 建立信任
    
- **长期**：愿景故事 → 塑造品牌
    
- **教程/培训**：教育用户，转化和留存
